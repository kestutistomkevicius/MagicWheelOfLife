SnapshotsPage receives `wheelId`, `userId`, and `categories` (the live localCategories from WheelPage) as props, OR fetches them internally via useWheel. Use the internal fetch approach (consistent with other pages — call useAuth + useWheel inside the page) and pass `localCategories` via a prop from WheelPage, since WheelPage already has the live slider state.

**Actually:** To avoid prop-drilling localCategories through the router, the simplest approach is: SnapshotsPage calls `useAuth()` and `useWheel(userId)` itself. For the "Save Snapshot" action, it uses `wheel` from useWheel and the `categories` array from the hook (which reflects the last-committed DB state, not the live slider state). This is acceptable for the snapshot page — users who want to snapshot their latest scores should commit the sliders first (the UI can display a note). This avoids introducing a shared state context for this phase.
