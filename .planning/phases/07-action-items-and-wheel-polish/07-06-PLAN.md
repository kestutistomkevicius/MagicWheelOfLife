**src/components/WheelChart.tsx changes:**

1. Update interface:
   ```typescript
   interface WheelChartProps {
     data: WheelChartPoint[]
     highlightedCategory?: string
     importantCategories?: string[]
   }
   ```
2. Inside the component, compute filtered datasets:
   ```typescript
   const importantData: WheelChartPoint[] = data.map(d => ({
     category: d.category,
     asis: (importantCategories ?? []).includes(d.category) ? d.asis : 0,
     tobe: 0,
   }))
   const highlightData: WheelChartPoint[] = data.map(d => ({
     category: d.category,
     asis: d.category === highlightedCategory ? d.asis : 0,
     tobe: 0,
   }))
   ```
3. Add two Radar components inside RadarChart (after existing Radars):
   ```tsx
   {(importantCategories ?? []).length > 0 && (
     <Radar
       name="Important"
       dataKey="asis"
       data={importantData}
       stroke="#b45309"
       fill="#b45309"
       fillOpacity={0.65}
       dot={false}
       legendType="none"
     />
   )}
   {highlightedCategory && (
     <Radar
       name="Highlighted"
       dataKey="asis"
       data={highlightData}
       stroke="none"
       fill="#fbbf24"
       fillOpacity={0.5}
       dot={false}
       legendType="none"
     />
   )}
   ```

Note: Recharts RadarChart uses the top-level `data` prop on RadarChart, not on individual Radar components, by default. To use different data per Radar, each Radar must have its own `data` prop.
