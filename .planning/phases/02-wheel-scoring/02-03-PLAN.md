const DEFAULT_CATEGORIES = [
  'Health', 'Career', 'Relationships', 'Finance',
  'Fun & Recreation', 'Personal Growth', 'Physical Environment', 'Family & Friends'

export type CreateWheelMode = 'template' | 'blank'

export interface UseWheelResult {
  wheel: WheelRow | null | undefined  // undefined = loading
  categories: CategoryRow[]
  loading: boolean
  error: string | null
  canCreateWheel: boolean
  createWheel: (mode: CreateWheelMode, userId: string) => Promise<WheelRow | null>

**useWheel(userId: string) logic:**
1. On mount, fetch user's profile (`profiles.tier`) and first wheel (ORDER BY created_at ASC LIMIT 1)
2. If wheel exists, fetch its categories (ORDER BY position ASC)
3. Compute `canCreateWheel`: true if tier='premium' OR wheel count is 0
4. Return all state + mutations

**createWheel(mode, userId):**
- INSERT into wheels with `{ user_id: userId, name: 'My Wheel' }`

- After creation, update local state (setWheel, setCategories) so the page re-renders without a full reload

**updateScore(categoryId, field, value):**
- `supabase.from('categories').update({ [field]: value, updated_at: new Date().toISOString() }).eq('id', categoryId)`
- Does NOT update local state — caller manages local state via setCategories before calling this

**addCategory:** Guard currentCount >= 12 → return { error: 'Maximum 12 categories reached' }. Otherwise INSERT with position = currentMaxPosition + 1 and score_asis = 5, score_tobe = 5.

**renameCategory:** If hasSnapshots → call onSnapshotWarning() and return (caller handles confirm/cancel flow). Otherwise call Supabase UPDATE directly.

**removeCategory:** Guard currentCount <= 3 → return { error: 'Minimum 3 categories required' }. If hasSnapshots → call onSnapshotWarning() and return. Otherwise call Supabase DELETE.

**Snapshot check (Phase 2 note):** In Phase 2, the `snapshots` table does not exist. The `hasSnapshots` parameter is passed in from the page level — the page will pass `false` in Phase 2 (hardcoded).
